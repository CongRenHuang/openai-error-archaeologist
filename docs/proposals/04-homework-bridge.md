# 提案 D(保底):作業翻譯官(Homework Bridge)— 移民家長共學橋樑

> 一句話:讓不會英文的媽媽,也能輔導孩子的英文作業。

## 1. 問題

移民家庭中,家長因語言隔閡無法參與孩子的學業——不是不想,是看不懂作業在教什麼。研究一致顯示家長參與度是學業成就的強預測因子,語言隔閡因此讓教育不平等代際傳遞。現有翻譯工具只翻字面,翻不出「這題在教什麼概念、家長該怎麼引導」。

## 2. 目標受眾

- **主要**:非在地語言母語的家長(美國西語/華語家庭、各國新住民家庭)。
- **次要**:學校家長聯絡窗口、課後輔導 NGO。

## 3. 解決方案

1. 家長拍照孩子的作業。
2. GPT-5.6 以家長的母語輸出三件事:**這題在教什麼概念**(不是逐字翻譯)、**孩子可能卡在哪**、**你可以怎麼引導**(提問腳本,不是答案)。
3. 刻意不給答案——給的是「家長版教師手冊」,保護孩子的學習過程。
4. 對話模式:家長用母語追問,AI 持續扮演「家長的教學教練」。

## 4. GPT-5.6 運用深度

| 能力 | 用法 |
|---|---|
| 多模態 | 作業照片理解(題型、科目、年級推斷) |
| 教學法轉譯 | 「教什麼概念 + 如何引導」的二階輸出,非字面翻譯 |
| 文化在地化 | 引導腳本適應家長的文化脈絡(例:數學直式算法各國不同) |
| 多語對話 | 家長母語的後續追問 |

技術深度為四案最淺——此為已知短板,以完成度與敘事補償。

## 5. MVP 切分

**做**:拍照 → 三段式家長指引(2 個語言:西語+華語);追問對話;5 個示範作業樣本。
**不做**:帳號、歷史紀錄、學校端、語音。

## 6. 八天里程碑

| 日期 | 交付 |
|---|---|
| 7/13 | 定案 + 腳手架 + 樣本收集 |
| 7/14–15 | 照片 → 三段式指引 pipeline(單語言) |
| 7/16–17 | 對話模式 + 第二語言 + UI 完整化 |
| 7/18–19 | 測試 + 文化在地化案例打磨;freeze |
| 7/20–21 | Demo、Devpost、提交 |

## 7. Demo 劇本(3 分鐘)

1. 鉤子:真實場景——孩子拿作業求助,媽媽看不懂,只能說「去問老師」。
2. live:拍照三年級數學作業 → 西語輸出「這題在教位值概念;別直接給答案,先問他…」→ 媽媽用西語追問。
3. 文化細節 wow:AI 提醒「墨西哥直式減法借位寫法與美國教的不同,孩子可能因此混淆」。
4. 影響力 + GTM。

## 8. 風險與緩解

| 風險 | 緩解 |
|---|---|
| 評分①技術深度吃虧 | 主打「教學法轉譯 + 文化在地化」的 prompt 設計深度;誠實定位 |
| 被歸類為「翻譯 app」 | demo 開場即對比:Google 翻譯逐字翻 vs 本產品的教師手冊式輸出 |
| 兩語言品質驗證人力不足 | 團隊自身中文母語可驗中文;西語找社群志願者抽驗 |

## 9. GTM 一頁

- **市場**:美國 K-12 學生 23% 來自移民家庭;各國新住民教育預算、學區家長參與計畫(Title I 家長參與經費)。
- **切入**:NGO/學區免費試點 → 學區采購 → 電信/銀行新住民服務的白牌合作。
- **定位**:四案中完成度最保險、敘事最動人,但技術深度天花板最低——僅在 A/B 案技術風險爆掉時降級採用。
