# 提案 C:閱讀處方箋(Reading Rx)— 讀寫障礙自適應閱讀器

> 一句話:不只換字型——AI 在語言層級為每一位讀寫障礙者重寫世界。

## 1. 問題

全球約 10% 人口有不同程度的讀寫障礙(dyslexia)。現有輔助工具停留在表層:特殊字型、底色、行距。但讀寫障礙的核心困難在語言處理層——音節解碼、長句工作記憶負荷、抽象詞彙。表層工具治標,語言層級的改寫才治本,而這正是 LLM 才做得到的事。

## 2. 目標受眾

- **主要**:有讀寫障礙的中小學生及其家長。
- **次要**:特教老師、資源班;成人讀寫障礙者(職場文件)。

## 3. 解決方案

1. 貼上任何網頁/PDF/作業文本。
2. GPT-5.6 依使用者的「閱讀檔案」做語言層級轉換:音節切分著色、長句拆短、抽象詞附具象例子、段落間插入理解檢核點。
3. 讀者答錯檢核點時,系統記錄錯誤模式(音韻型?語法型?詞彙型?)→ 更新閱讀檔案 → 下一篇文本的改寫策略隨之調整。
4. 家長/老師端:閱讀檔案報告,顯示孩子的困難模式與進步曲線。

## 4. GPT-5.6 運用深度

| 能力 | 用法 |
|---|---|
| 語言學分解 | 音節切分、句法簡化(保留原意的受控改寫) |
| 個人化改寫 | 依閱讀檔案參數(音韻/語法/詞彙弱點)調整改寫策略 |
| 理解檢核生成 | 針對段落自動生成診斷性問題(區分解碼失敗 vs 理解失敗) |
| 錯誤模式歸因 | 從答錯紀錄推斷困難類型,structured output 更新檔案 |

## 5. MVP 切分

**做**:網頁文字貼上 → 改寫閱讀器(3 種呈現層級可切換);理解檢核 + 閱讀檔案雛形(3 個參數);家長報告一頁。
**不做**:PDF 解析、瀏覽器外掛、TTS 對齊朗讀、多語言。

## 7. Demo 劇本(3 分鐘)

1. 鉤子:同一段課文,一般呈現 vs 讀寫障礙者眼中的呈現(模擬動畫)。
2. live:貼上真實課文 → 三層級改寫即時切換 → 答錯檢核點 → 下一段改寫策略當場改變。
3. 家長報告 + 影響力數字(10% 人口)+ GTM。

## 8. 風險與緩解

| 風險 | 緩解 |
|---|---|
| 創意分:品類已存在,恐被視為改良 | 敘事強調「語言層級 vs 表層工具」的代差;live 自適應是差異化 |
| 特教專業性遭質疑 | 改寫策略對映文獻(結構化教學法/Orton-Gillingham 原則),README 附來源 |
| 自適應引擎 8 天做不深 | 收斂為 3 參數規則引擎 + GPT-5.6 歸因,誠實標注 roadmap |
| 改寫失真(意思跑掉) | 受控改寫 + 原文對照模式一鍵切換 |

## 9. GTM 一頁

- **市場**:特教科技(assistive tech)采購有政策預算(IDEA/IEP 相關經費,各國同類);家長自費意願高。
- **切入**:B2C 家長訂閱 → 學校資源班授權 → 出版社無障礙版本 API。
- **評審對位**:教育平權敘事最貼 VP of Education 的公開關注領域;影響力分數天花板最高。
