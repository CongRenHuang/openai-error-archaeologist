# 提案 A(首推):錯誤考古學家(Error Archaeologist)— 錯因假說診斷引擎

> 一句話:**AI 不猜學生腦中想什麼;它找出最可能的錯因、指出證據在第幾步,再生成一題能區分候選錯因的鑑別問題。**
>
> 版本 4(2026-07-14):依 PM 修正報告(improvement-measures.md)全面修訂。核心轉向:「AI 判定錯因」→「AI 提出可驗證的診斷假說」。砍 Python taxonomy、砍未校準信心分數、砍「0 行 code 換客戶」;微課降為 P2;新增鑑別問題(P1)、24–30 份盲測集、教師訪談計畫;修正 NAEP 表述與合規聲明。

---

## 第一部:這個問題值不值得解

## 1. 問題

老師改作業只看對錯,看不出錯誤背後的原因。同一個錯誤答案背後可能藏著多種不同成因——例如 `-3(x - 2) = -3x - 6` 可能是分配律迷思、負號規則錯誤、粗心或抄寫疏忽,各需完全不同的處置。錯因分析無法規模化,美國砸了千億美元的學習恢復政策因此缺了中間一層:篩得出「誰落後」,補得起「開課教學」,唯獨「為什麼落後」無人回答。

**誠實前提**(本版核心修正):單份作業**無法唯一辨識**錯因——同一錯誤可來自迷思、粗心、抄寫、題意誤讀。因此本產品不做「AI 宣判」,做「AI 提出候選假說 + 證據 + 一題鑑別問題」,由老師確認、由學生的下一個回答收斂。這不是退讓,是比「AI 讀心」更科學也更可信的產品定位。

## 2. 可驗證性:痛點的規模與強度(原則 01)

**規模——美國教育的頭號未解問題**

- NAEP 2024(美國官方「Nation's Report Card」):八年級數學 **28% 達 Proficient、61% 達 Basic、39% 低於 Basic** [1]。注意:NAEP Proficient 是高標,**不等於年級達標**——本提案不使用「72% 未達精熟」這種易誤導的說法;低於 Basic 的近四成學生才是補救政策的目標人群。
- 疫情五年後**全國仍未恢復**:沒有任何一州的八年級成績回到 2019 疫情前水準 [2]。
- 經濟代價:Hanushek (2023) 估計未補救的成績下滑總體經濟損失達 **31 兆美元** [3](demo 少講此數字,見 §11——評審要的是具體可信,不是天文數字)。

**強度——政府已證明願意付費,但主力解法無法規模化**

- 聯邦投入 **$190B ESSER 紓困經費**,高劑量家教(HDT)為首要策略;全美 66% 公立學校開辦 [3][4]。
- HDT 成本 **$1,200–$2,000/學生/年**(MDRC 2025 實測)[4];有效的密集家教方案僅覆蓋 1–2% 合資格學生 [3]。
- 規模化後效果衰減至 0.06–0.09 SD,遠低於小規模研究的 0.18–0.40 SD [4][6]。

**教育學證據——誠實表述其邊界**

- Russell, O'Dwyer & Miranda (2009,905 學生/44 教師隨機叢集試驗):給教師「迷思診斷報告 + 對症教材」的完整介入顯著提升代數能力(+0.13 SD);只給總分的組別效果最差 [7]。
- **該研究支持的是**:特定診斷工具 + 特定教材的組合可能有效。**它不支持**:「LLM 從手寫推錯因再生成微課,同樣有效」。本提案引用它作為 **intervention hypothesis 的依據**——「診斷先於補救」這個方向有實證基礎——而非宣稱複製其效果。我們自己的有效性證據來自 §10 的盲測。

## 3. 需求驅動:LLM 之前的世界(原則 04)

| 世代      | 美國既有解法                                                | 失效點                                                                                                        |
| --------- | ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| 人力      | 高劑量家教(1 對 1–2,診斷發生在家教腦中)                     | $1,200–2,000/生/年;有效方案僅覆蓋 1–2% 學生 [3][4][5]                                                         |
| 規則/題庫 | 適應性選擇題診斷:i-Ready Diagnostic、NWEA MAP、DAAS [7]     | 每概念需專家預編誘答選項;只能出**選擇題**,讀不了學生自然手寫的解題過程;輸出「落後幾級」而非「錯誤規則是什麼」 |
| 傳統 ML   | 認知診斷模型(CDM/DINA)[8]、ITS(Cognitive Tutor/ASSISTments) | 依賴預先設計的 Q-matrix 與封閉題型;ITS 只在自家系統內診斷,學生日常紙本作業碰不到                              |

**LLM 是質變,不是量變**:

1. **讀自由形式手寫過程**——學生真實的紙本作業,前三代技術原理上做不到。
2. **免預編題庫**——新單元邊際成本趨近零。
3. **證據定位**——把候選錯因錨定到「學生自己寫的第幾步」,需要對推導過程的語意理解。
4. **即時生成鑑別問題**——針對兩個候選錯因,生成一題「答 A 則是錯因一、答 B 則是錯因二」的區分題。題庫模式做不到,因為它無法預知會出現哪對候選錯因。

**模型鎖定注意**:目前公開 API 主力為 GPT-5.5(已支援 image input 與 structured outputs)。競賽若提供 GPT-5.6 preview,開賽日以官方公布的實際 model ID 為準寫死於設定檔,不假設 `gpt-5.6` 字串存在;pipeline 對模型版本無耦合(§5)。

## 4. 解決方案:假說 → 證據 → 鑑別 → 確認

1. 老師拍照上傳學生手寫作業(demo 主線用預驗證樣本 + live 上傳加分)。
2. 模型視覺理解手寫過程,輸出**固定 structured output**:

```json
{
  "observed_error": "將 -3(x-2) 展開為 -3x-6",
  "candidate_misconceptions": [
    { "id": "NEG_DIST", "label": "負號未分配到第二項", "confidence_band": "High" },
    { "id": "SIGN_RULE", "label": "負負得正規則誤用", "confidence_band": "Medium" }
  ],
  "supporting_evidence": "第 2 步:-3·(-2) 寫成 -6",
  "alternative_explanation": "亦可能為抄寫疏忽(僅此一步出錯,其餘步驟正確)",
  "verification_question": "展開 -2(a - 5) = ?",
  "abstain_reason": null
}
```

3. **鑑別問題**(本產品最有創意的一步):AI 生成一題能區分兩個候選錯因的題目;學生回答後,診斷收斂或修正。
4. 老師一鍵確認/否決 → 確認後才進班級熱力圖與後續處置。
5. 對症練習(P2):時間允許則由確認後的錯因觸發模板化練習;**不承諾即時自由生成微課**。

**信心度規則**(取代 v3 的 0.7 固定門檻——LLM 自報 0.83 不代表 83% 正確):

- **High**:關鍵步驟直接支持,且無競爭解釋
- **Medium**:存在兩個合理錯因
- **Abstain**:影像或推導不足,不出診斷,說明缺什麼
- MVP 不顯示假精準小數;正式版以 calibration set 設數值門檻(roadmap)。

**GPT-5.6 運用深度(Build Week 評分①)**:多模態手寫理解(非 OCR,懂步驟間推導關係)× 溯因推理(abductive:從錯誤結果反推候選規則)× structured output(固定 schema 可聚合、可 eval)× **鑑別問題生成**(針對候選假說對即時出題)× abstain 機制。五層疊加,且「假說—證據—鑑別—收斂」是完整推理迴圈,不是單發 prompt。

## 5. 可重用性:框架與 AI 解耦(原則 02)

```
┌─────────────────────────────────────────────────────┐
│  錯因考古 pipeline(領域無關核心)                       │
│  產出物 → 候選假說 → 證據定位 → 鑑別收斂 → 教師確認 → 聚合 │
└─────────────────────────────────────────────────────┘
        ↑ 可插拔               ↑ 可插拔          ↑ 可插拔
   AI 推理引擎層          Taxonomy 設定檔      輸入/輸出 adapter
  (model ID 設定檔化)    (JSON,按領域置換)   (拍照;其餘 roadmap)
```

**把 LLM 抽掉,框架仍成立**:「收集產出物 → 歸因候選錯誤 → 鑑別驗證 → 聚合成教學決策」正是特教與家教領域的人工錯誤分析 SOP——LLM 讓這套方法論第一次可規模化。

跨領域重用(程式 bug 迷思、語言學習母語遷移、醫學臨床推理)是**架構事實 + roadmap**,不是 demo 內容——v3 的 Python taxonomy 彩蛋已砍:它無法真正證明跨域效度,反而稀釋產品故事。MVP 用單一份外置 algebra taxonomy JSON 證明解耦即可。

## 9. 失敗成本與風險設計(原則 07)

誤診後果:老師據錯誤診斷補救 → 浪費教學時間、學生被貼錯標籤。中風險場域,四道防線:

- **假說而非判決**:輸出永遠是 candidate_misconceptions[] + alternative_explanation,UI 文案不出現「該生的錯因是」斷言句。
- **Teacher-in-the-loop**:老師確認/否決後,結果才進熱力圖、練習才發給學生。AI 是幕僚,不是裁判。
- **Abstain 機制**:影像不清或推導不足 → 不硬出診斷,顯示缺什麼(「需要看到第 3 步之後的計算」)。
- **證據強制**:每個假說必附「第幾步露餡」,老師 10 秒可自行核驗——可否證的診斷才是負責任的診斷。
- 技術風險:手寫辨識不穩 → demo 主線用預驗證樣本集;「prompt 包裝」質疑 → 外置 taxonomy + 盲測數據公開(§10)。

## 10. 資料可取得性與有效性驗證(原則 08)

**資料來源**

| 需求         | 來源                                                     | 取得路徑                              |
| ------------ | -------------------------------------------------------- | -------------------------------------- |
| 題目         | NAEP 公開釋出題(public domain)+ 自編                     | 已公開,零成本 [1]                     |
| 手寫錯題樣本 | **團隊自製擬真樣本**(見下方盲測集設計)                    | 8 天內可完成;不需真實學生資料          |
| Taxonomy     | 已發表代數迷思文獻(DAAS 三概念 [7]、錯誤分析研究)         | 公開文獻,README 附引                  |
| 真實學生資料 | MVP 不使用                                               | 產線階段:學區協議 + 去識別化,roadmap  |

**合規聲明**(修正 v3 的過度承諾):不寫「無 FERPA/COPPA 風險」。正確表述:**Demo 僅允許合成或去識別樣本;明文禁止上傳真實學生資料;影像不持久化,自動刪除**。評審若上傳含姓名/字跡的真實作業,系統顯示去識別提醒。

**盲測集(取代 v3 的 10 樣本自測——自編自測是 circular validation,會被評審一句戳破)**:

- **24–30 份樣本、8–10 種錯因**;每種含典型、模糊、無錯、不可判定四類案例;含「相同答案、不同錯因」與「粗心/抄錯」對照組。
- 兩位標註者**先獨立標註、封存答案**,之後才調 prompt;模型不看答案。
- **保留 ≥8 份完全未調參的 holdout**。
- 報告:錯因分類 macro-F1、證據步驟正確率、abstain 正確率、教師確認耗時、失敗案例。
- 即使只有 75%,誠實展示錯誤與安全機制,可信度仍高於「10 個內建樣本全過」。

## 11. Demo 可視性:怎麼演(原則 09)

產出是**可互動網站**(Vercel 公開 URL,評審可自己玩)。主線 3 分鐘:

1. **0:00** 鉤子:兩份**同答案、不同思路**的手寫作業並排——「答案一樣錯,原因完全不同」。
2. **0:20** 上傳作業照片。
3. **0:50** 模型提出**兩個候選錯因假說**,證據高亮在學生自己寫的第 2 步。
4. **1:20** 生成一題**鑑別問題**——這題答 A 是錯因一、答 B 是錯因二。
5. **1:50** 學生回答 → 其中一個錯因被排除,診斷收斂。
6. **2:15** 老師一鍵確認 → 班級熱力圖更新:「全班 40% 同一迷思,下節課教這個」。
7. **2:40** 展示 **holdout 盲測結果與 abstain 案例**——包含模型答錯的案例,誠實呈現。

影響力敘事用「教師訪談 + 盲測數據 + 節省的分析時間」,少講 $31T。評審互動 2 分鐘:點按其餘樣本、自行上傳、看 abstain 欄與否決流程。

四項評分對應:技術實作(候選假說 + 證據定位 + structured output + 鑑別題 + abstain)、設計(完整教師決策流)、影響力(訪談 + 盲測 + 省時)、創意(AI 不宣判,提出可驗證假說再用鑑別題收斂)。

## 12. 護城河(原則 10)

「GPT 包 UI」一週可複製;以下不可:

1. **經驗證的 taxonomy × 盲測語料飛輪**:每次老師確認/否決 = 一筆標註資料;盲測集本身即私有評測資產——用得越多,診斷越準。
2. **領域 know-how 沉澱**:把 40 年迷思文獻編碼成可執行 taxonomy 與判準 + 鑑別題生成邏輯,是教育測量 × 工程的交叉壁壘。
3. **工作流卡位**:嵌在「篩選(i-Ready)→ **診斷(我們)** → 補救(家教/開班)」缺口層。
4. 模型不是護城河——架構上模型層為設定檔可替換(§5),價值沉澱在框架、評測集與確認資料。
