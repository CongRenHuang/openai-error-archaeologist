# 提案 A(首推):錯誤考古學家(Error Archaeologist)— 迷思概念診斷引擎

> 一句話:AI 不告訴你答案,AI 告訴你「為什麼你會這樣錯」。
>
> 版本 3(2026-07-13):依十項檢核原則完整重構。資料來源收斂至**美國**——評審五位皆為 OpenAI(美國)人員,Leah Belsky(VP of Education)長期深耕美國教育市場;美國疫後學習損失是其社群最有感的敘事。所有事實句附來源,見文末文獻清單。

## 第一部:這個問題值不值得解

## 1. 問題

老師改作業只看對錯,看不出錯誤背後的原因。同一個錯誤答案背後可能藏著多種不同的迷思概念(misconception)——例如 `-3(x - 2) = -3x - 6` 可能是分配律不完整、負號規則錯誤、或抄寫疏忽,三者需要完全不同的補救教學。錯因分析無法規模化,美國砸了千億美元的學習恢復政策因此缺了中間一層:篩得出「誰落後」,補得起「開課教學」,唯獨「為什麼落後」無人回答。

## 2. 可驗證性:痛點的規模與強度(原則 01)

**規模——這是美國教育的頭號未解問題**

- NAEP 2024(國家教育進展評估,美國官方「Nation's Report Card」):八年級數學僅 **28% 學生達精熟(Proficient)**,**近 40% 連 Basic 都不到**(這些學生「可能無法用相似形求三角形邊長」)[1][2]。
- 疫情五年後**全國仍未恢復**:沒有任何一州的八年級成績回到 2019 疫情前水準;參與評估的大型學區無一恢復 [2]。
- 經濟代價:Hanushek (2023) 估計,未補救的成績下滑將使受影響學生終身收入減少 2–9%,總體經濟損失達 **31 兆美元** [3]。

**強度——政府已證明願意付費,但主力解法無法規模化**

- 聯邦為此投入 **$190B ESSER 紓困經費**,教育部長點名「高劑量家教(high-dosage tutoring, HDT)」為首要策略;全美 66% 公立學校開辦 HDT [3][4]。
- 但 HDT 成本 **$1,200–$2,500/學生/年** [4][5];哈佛/CALDER 八大學區研究:少數有效的密集家教方案,**僅覆蓋 1–2% 的合資格學生**——有效但撐不起規模 [3]。
- 規模化後效果衰減:疫後大規模實施的 HDT 效果僅 0.05–0.09 SD,遠低於小規模黃金研究的 0.18–0.40 SD(Guryan et al., AER)[4][6]。

**教育學證據——「先診斷、再對症」是被 RCT 驗證的正解**

- Russell, O'Dwyer & Miranda (2009,美國,905 學生/44 教師四組隨機叢集試驗):給教師「迷思診斷報告 + 對症教材」的完整介入顯著提升代數能力(+0.13 SD, p<.05);只給總分的組別效果最差 [7]。
- 診斷本身有量化價值:同一批答錯的學生,錯因分布完全不同——沒有錯因,補救就是亂槍打鳥。

> 收斂:規模 = 全美 ~72% 八年級生數學未達精熟 [1];強度 = 政府每年十億級預算在買「補救」,而其瓶頸(診斷層缺失 + 家教無法規模化)有官方研究背書。

## 3. 需求驅動:LLM 之前的世界(原則 04)

| 世代      | 美國既有解法                                                | 失效點                                                                                                        |
| --------- | ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| 人力      | 高劑量家教(1 對 1–2,診斷發生在家教腦中)                     | $1,200–2,500/生/年;有效方案僅覆蓋 1–2% 學生 [3][4][5]                                                         |
| 規則/題庫 | 適應性選擇題診斷:i-Ready Diagnostic、NWEA MAP、DAAS [7]     | 每概念需專家預編誘答選項;只能出**選擇題**,讀不了學生自然手寫的解題過程;輸出「落後幾級」而非「錯誤規則是什麼」 |
| 傳統 ML   | 認知診斷模型(CDM/DINA)[8]、ITS(Cognitive Tutor/ASSISTments) | 仍依賴預先設計的 Q-matrix 與封閉題型;ITS 只在自家系統內診斷,學生日常紙本作業碰不到                            |

**LLM 是質變,不是量變**:

1. **讀自由形式手寫過程**——學生真實的紙本作業,前三代技術原理上做不到(不是貴,是不能)。
2. **免預編題庫**——新單元邊際成本趨近零;題庫模式每個概念都要專家迴圈。
3. **證據定位**——把錯因錨定到「學生自己寫的第幾步」,需要對推導過程的語意理解。

一句話:LLM 之前,「晤談級診斷」與「題庫級規模」不可兼得;本提案讓前者跑在後者的規模上。若 LLM 只是讓既有題庫便宜 30%,不值得做——它做到的是類別缺失的能力。

## 4. 解決方案

1. 老師拍照上傳學生手寫作業(單張或整疊)。
2. GPT-5.6 視覺理解手寫過程 → **逆向推理學生的思考路徑**,重建「學生腦中的錯誤規則」。
3. 對映迷思分類法(taxonomy 外置 JSON,見 §5),輸出:錯因標籤、信心度、證據步驟。
4. 生成對症微課(講解 + 2 題對症練習)——複製 DAAS RCT 中唯一顯著有效的「診斷+對症教材」完整介入 [7]。
5. 老師儀表板:全班迷思熱力圖——「全班 40% 卡在負號分配」,下節課直接教這個。

**GPT-5.6 運用深度(Build Week 評分①)**:多模態手寫理解(非 OCR,要懂步驟間推導關係)× 逆向推理(abductive:從錯誤結果反推錯誤規則)× structured output(taxonomy JSON 可聚合)× 條件生成(微課隨診斷變化)。四層疊加,非平凡實作。

## 5. 可重用性:框架與 AI 解耦(原則 02)

```
┌─────────────────────────────────────────────────────┐
│  錯因考古 pipeline(領域無關核心)                       │
│  產出物 artifact → 逆向推理 → 分類法對映 → 聚合儀表板    │
└─────────────────────────────────────────────────────┘
        ↑ 可插拔               ↑ 可插拔          ↑ 可插拔
   AI 推理引擎層          Taxonomy 設定檔      輸入/輸出 adapter
  (GPT-5.6,可替換)      (JSON,按領域置換)   (拍照/LMS/API)
```

**把 LLM 抽掉,框架仍成立**:「收集產出物 → 歸因錯誤規則 → 對映分類法 → 聚合成教學決策」正是特教與家教領域的人工錯誤分析 SOP——LLM 只是讓這套本來有效的方法論第一次可規模化。此為原則 04 判準的正面證明。

換 taxonomy 即換領域:程式教育(bug 迷思)、語言學習(母語遷移錯誤)、醫學教育(臨床推理偏誤)、企業訓練(SOP 誤解)。MVP 內建第二份 Python taxonomy 實證此點(§8)。

## 6. 通用性:部署適應彈性(原則 03)

**判準回答:換一家客戶,改多少 code?——0 行核心 code,只換設定與 adapter。**

- Taxonomy = JSON 設定檔:換州課綱(Common Core ↔ 德州 TEKS)= 換檔案。
- 輸入 adapter:無系統的學校用拍照;用 Google Classroom/Canvas 的走 LMS 匯出;學區級整合走美國 K-12 既有標準介面(**LTI 1.3、OneRoster、Ed-Fi**)與 SSO(Clever/ClassLink)。
- 輸出 adapter:教師儀表板 / CSV / API,對接學區既有數據倉。
- 定位為既有評估系統(i-Ready、NWEA MAP)的**錯因層擴充**而非替代:它們篩「誰落後」,我們答「為什麼」——採購上是加購件,不是換系統。

## 7. 商業價值:價值鏈三角色(原則 05)

| 角色          | 是誰                                                                                                                                                                                                 | 說明                                                                                                                                                                                                                                                      |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **User**      | 老師(每天用)、學生(收到對症微課)                                                                                                                                                                     | 使用者不付錢                                                                                                                                                                                                                                              |
| **Payer**     | 學區/州政府                                                                                                                                                                                          | 採購動機:Title I-A 每年 **$18.4B** 可用於補救性服務 [9];田納西州已把家教錢**寫進州經費公式**($500/四年級生/年)[5];德州立法強制對州測未過學生提供補救但**未撥款** [5]——無資金的強制令 = 對低成本診斷工具的剛性需求;NAEP 恢復進度是各州教育廳的公開 KPI [2] |
| **Incumbent** | 順風方:補教/家教機構(Saga 等)、教材商——診斷報告幫他們把 $1,200/生的家教錢花在刀口上,策略=賣鏟子(API/白牌)。擋路方:診斷測驗商(i-Ready/NWEA 的題庫商業模式被繞過)——預期以信效度質疑反擊,應對見 §9、§12 |

Payer 與 User 分離已說明:學區買單的理由是「用診斷讓既有補救預算(家教/開班)的命中率變高」,是預算效率敘事,不是新增支出敘事。

## 9. 失敗成本與風險設計(原則 07)

誤診後果:老師據錯誤診斷做補救 → 浪費教學時間、學生被貼錯標籤。中風險場域,必須設計:

- **Teacher-in-the-loop**:診斷結果先到老師,老師一鍵確認/否決後,微課才會發給學生。AI 是幕僚,不是裁判。
- **信心門檻**:信心 < 0.7 的診斷自動進「不確定」欄,附「建議口頭確認」提示,不出現在熱力圖統計。
- **證據強制**:每個診斷必附「第幾步露餡」,老師 10 秒可自行核驗——可否證的診斷才是負責任的診斷。
- 技術風險:手寫辨識不穩 → demo 主線用預驗證樣本集,live 上傳為加分演示;「prompt 包裝」質疑 → 外置 taxonomy 架構 + 10 樣本驗證集準確率公開。

## 11. Demo 可視性:5 分鐘怎麼演(原則 09)

產出不是報告,是**可互動網站**(Vercel 公開 URL,評審可自己玩):

1. **0:00–0:30** 鉤子:三份同題不同錯的手寫作業並排 +「全美 40% 八年級生數學低於 Basic,$190B 花下去,沒人知道每個學生*為什麼*錯」[1][3]。
2. **0:30–1:30** live 上傳照片 → 30 秒內:「該生將 -3(x-2) 處理為 -3x-6:負號未分配到第二項。證據:第 2 步」+ 信心度。
3. **1:30–2:30** 老師一鍵確認 → 對症微課即時生成;切熱力圖:「全班 40% 同一迷思,下節課教這個」。
4. **2:30–3:00** 換 Python taxonomy 設定檔重跑 →「這不是數學 app,是錯因考古框架」。
5. **3:00–5:00**(留給評審互動)點按其餘 9 個樣本、自行上傳、看不確定欄與否決流程。

每個評分維度都有對應畫面:技術深度(§4 四層)、完整產品(全流程可點)、影響力(§2 數據)、創意(反向診斷 + taxonomy 熱插拔)。

## 12. 護城河(原則 10)

「GPT 包 UI」一週可複製;以下不可:

1. **經驗證的 taxonomy × 錯題語料飛輪**:每次老師確認/否決診斷 = 一筆標註資料,累積成私有評測集與微調語料——用得越多,診斷越準,後進者沒有這個迴圈。
2. **領域 know-how 沉澱**:把 40 年迷思概念文獻編碼成可執行的 taxonomy 與判準,是教育測量專業 × 工程的交叉壁壘,不是 prompt。
3. **工作流卡位**:嵌在「篩選(i-Ready)→ **診斷(我們)** → 補救(家教/開班)」的缺口層,兩端都是我們的資料上下游,替換成本隨整合加深。
4. 模型不是護城河(GPT-5.6 人人可用)——所以架構上把模型層設計成可替換(§5),價值沉澱在框架與資料。
