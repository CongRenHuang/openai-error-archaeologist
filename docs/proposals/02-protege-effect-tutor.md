# 提案 B(備選):教學相長(Teach the Machine)— 學生教 AI 的反向家教

> 一句話:AI 不當老師,AI 當那個聽不太懂的同學——你得把它教會。

## 1. 問題

被動聽講的知識留存率極低,而「教別人」是留存率最高的學習方式(protégé effect / learning-by-teaching,有教育心理學研究支持)。但現實中學生找不到隨時願意被教、且困惑點恰好對準課綱的「同學」。市面上所有 AI 教育產品都讓 AI 當老師——這條路已擠滿「AI 家教 chatbot」。

## 2. 目標受眾

- **主要**:國高中生,考前自我檢驗「我是真懂還是背起來」。
- **次要**:老師——把「教會 AI」當作業,收 AI 產出的理解度報告。

## 3. 解決方案

1. 學生選一個概念(如「光合作用」),AI 化身「小迷糊」——一個依課綱校準困惑點的裝不懂同學。
2. 學生用自己的話解釋;小迷糊會誤解、追問、舉錯誤例子(這些困惑點正是該概念的常見迷思)。
3. 學生必須修正小迷糊的誤解,直到它「聽懂」並能正確複述、舉一反三。
4. 結束後產出「理解度報告」:哪些追問你答不出來 = 你自己的理解漏洞。
5. 語音模式(realtime voice):口頭教學,更接近真實教同學的體驗。

## 4. GPT-5.6 運用深度(評分①核心)

| 能力 | 用法 | 非平凡之處 |
|---|---|---|
| 校準式 persona | 「裝笨」需精確控制:困惑點來自該概念的真實常見迷思,不是隨機犯蠢 | 展示可控性 prompt 工程 + 課綱結構化知識 |
| 雙軌評估 | 對話同時,第二軌以 rubric 評估學生解釋品質(準確/完整/類比恰當) | 一次對話,兩個模型角色並行 |
| 判定「被教會」 | AI 需判斷自己何時該「懂」——太快沒挑戰性,太慢挫折感 | 教學節奏建模,非平凡的狀態機 |
| Realtime voice | 口頭教學模式 | demo wow 因子 |

## 5. MVP 切分

**做**:3 個示範概念(代數分配律/光合作用/供需法則,展示跨科通用);文字對話完整流程;理解度報告;語音模式(若 realtime API 順利,否則降級為 TTS)。
**不做**:全課綱覆蓋、帳號系統、老師端班級管理、遊戲化。

## 6. 八天里程碑

| 日期 | 交付 |
|---|---|
| 7/13 | 定案 + 腳手架;3 概念的迷思庫設計 |
| 7/14–15 | 小迷糊對話引擎(persona + 被教會判定)單概念走通 |
| 7/16–17 | 雙軌 rubric 評估 + 理解度報告 + UI 完整流程 |
| 7/18–19 | 語音模式 + 擴至 3 概念 + 測試;7/19 晚 freeze |
| 7/20–21 | Demo 影片、Devpost、提交 |

## 7. Demo 劇本(3 分鐘)

1. **0:00–0:30** 鉤子:「留存率:聽講 10%,教別人 90%。問題是——你要教誰?」
2. **0:30–1:45** live:學生解釋光合作用,小迷糊追問「所以植物晚上會把氧氣吃回去嗎?」學生語塞 → 修正解釋 → 小迷糊複述成功。
3. **1:45–2:30** 理解度報告:「你在『暗反應』的追問卡住了」——AI 沒教你,但你發現了自己不懂的地方。
4. **2:30–3:00** 語音模式 10 秒片段 + 架構 + GTM。

## 8. 風險與緩解

| 風險 | 緩解 |
|---|---|
| 「裝笨」失準(太蠢或太聰明) | 迷思庫驅動而非自由發揮;每概念人工校驗 10 輪對話 |
| 教學有效性難以在 demo 量化 | 理解度報告即量化證據;引用 protégé effect 文獻 |
| 影響力論證比 A 案抽象 | 主打「考前自檢」具體場景,不談抽象學習理論 |
| realtime voice 整合超時 | 降級方案:文字 + TTS,voice 只做 demo 片段 |

## 9. GTM 一頁

- **市場**:自主學習者與考試準備市場;老師端「教會 AI」作為新型作業形態。
- **切入**:B2C 免費三概念 → 訂閱解鎖全課綱 → B2B2C 學校授權。
- **差異化**:全市場 AI 都在當老師,只有我們讓 AI 當學生——類別開創者敘事。
- **與 A 案綜效**:可作為 A 案「診斷→補救」之後的「驗證」環節,兩案可合併為一條學習閉環故事。
