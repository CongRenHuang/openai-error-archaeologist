### 📝 參賽規則 (Rules)

* **參賽形式**：開放個人或組隊參賽，每支隊伍最多 **4 名**成員。
* **程式碼限制**：參賽者可以使用開源函式庫（Open source libraries），但**嚴禁使用專屬/私有程式碼（Proprietary code）**。
* **學生專屬獎項**：若欲角逐「最佳大學生作品（Best university submission）」類別，團隊中的**所有成員都必須是學生**身分。

### 🎯 評分重點與標準 (Judging Criteria)

評委將根據以下幾個核心維度來評估參賽作品：

**1. 綜合評分項目：**

* **設計與體驗**：使用者體驗設計 (UX)、使用者介面 (UI)、系統的可用性 (Usability)。
* **創新與創意**：專案的創新性 (Innovation)、創造力 (Creativity)。
* **實用性與影響**：能否有效改善既有的工作流程、案例研究範例的完整度。
* **商業與呈現**：產品展示的成果 (Product Demonstration)、進入市場策略 (Go to market strategy)。

**2. 各主題的特定評分重點：**

* **教育主題 (Education)**：是否能提供多樣化學習者（如：閱讀障礙者）的輔助工具、學生評量工具，或是協助教師備課的工具。
* **生產力主題 (Productivity)**：開發能改善工程師日常工作的 Visual Studio Code 擴充功能，例如：自動除錯、自動程式碼生成或補全功能。
* **Discord 生產力主題**：開發一款能夠改善日常任務或開發者工作流程的 Discord 機器人。
* **音樂與藝術主題 (Music, and the Arts)**：為創意軟體或平台開發外掛程式（Plugins）或整合 API，包含 Adobe 產品線（XD、Photoshop、Premiere Pro、After Effects、Illustrator、Substance），以及 Spotify API、Podcast API 或 Ableton Live 等。

* **專案簡介**：運用 GPT-3 技術自動生成字彙測驗。
* **團隊成員**：Jonny Kalambay、Bai Li

* **團隊成員**：Marco Fernandez、ojasviG Gupta、Sinan Robillard

* **[Kenni Velez Records AI INC](https://devpost.com/software/kenni-velez-records-ai-inc)**

* **團隊成員**：Kenni Velez Records AI INC
